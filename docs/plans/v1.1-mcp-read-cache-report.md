# v1.1 MCP Read-Path Cache Report

Date: 2026-03-04  
Scope: `#129 v1.1: MCP read-path caching with permission-safe keys`

## Delivered

- Added MCP read-cache support in `Waaseyaa\Mcp\McpController` for read-heavy tools:
  - `search_entities`
  - `search_teachings`
  - `ai_discover`
  - `traverse_relationships`
  - `get_related_entities`
  - `get_knowledge_graph`
- Implemented deterministic key contract:
  - hash of `{contract_version, tool, normalized_arguments, account_context}`
  - account context includes authentication state, account ID, and sorted roles
- Implemented transparent cache read/write path with unchanged tool payload contract.
- Added cache tagging for targeted invalidation:
  - `mcp_read`
  - `mcp_read:contract:v1.0`
  - `mcp_read:tool:{tool}`
  - entity tags from request/payload (`mcp_read:entity:{type}`, `mcp_read:entity:{type}:{id}`)
- Wired MCP cache bin in HTTP kernel:
  - new DB-backed bin/table: `mcp_read` / `cache_mcp_read`
  - `McpController` now receives this cache backend in the `mcp.endpoint` route path
- Added kernel invalidation listeners:
  - tag-aware invalidation on entity save/delete
  - non-tag fallback to `deleteAll()`

## Permission Safety Outcome

- Cache partitioning now prevents response reuse across different account contexts.
- Shared-key leakage between privileged and restricted callers is blocked by account-scoped key dimensions.
- Anonymous and authenticated contexts are explicitly separated.

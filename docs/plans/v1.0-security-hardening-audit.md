# v1.0 Security + Access-Control Hardening Audit

Date: 2026-03-04  
Scope: `#121 v1.0: Security and access-control hardening audit`

## Findings and Disposition

1. **Finding:** workflow/public visibility normalization was duplicated across multiple surfaces (`SearchController`, `EntityEmbeddingListener`, `SemanticIndexWarmer`, `RelationshipTraversalService`, MCP discovery checks, SSR discovery checks).  
   **Risk:** semantic drift could make draft/review/archived content visible on one surface while hidden on another.  
   **Disposition:** **Resolved** by introducing shared primitive `Waaseyaa\Workflows\WorkflowVisibility` and refactoring all identified consumers to use it.

2. **Finding:** regression coverage did not explicitly assert cross-surface consistency when `status` is truthy but `workflow_state` is non-public (e.g., `review`).  
   **Risk:** permission-edge ambiguity where `status`-only checks leak non-public workflow states.  
   **Disposition:** **Resolved** by adding `tests/Integration/Phase15/AccessVisibilityConsistencyIntegrationTest.php`.

## Security Assumptions and Invariants

- `workflow_state` is authoritative for node visibility when present.
- `status` is fallback-only for node visibility and primary for non-node entities that expose status.
- Public discovery/search surfaces must never expose non-`published` node state.
- Access-handler allowance does not override workflow public visibility rules for public surfaces.

## Residual Risks (Accepted)

- Policy mapping still depends on deployment-specific permissions/roles and can only be fully validated in application repos (tracked in Minoo posture docs).
- Relationship visibility for unknown entity types without `status` remains permissive by design (treated as public); this is required for generic graph interoperability but should be reviewed per deployment.
- Manifest optimization warnings (`composer dump-autoload --optimize`) remain operational, not security-blocking.

## Result

Hardening acceptance criteria are satisfied for framework scope: findings triaged, core gaps resolved, cross-surface semantics unified, and risk register updated.
